Transition

Introduction to Transitions

CSS Transitions allow us to specify how properties will change over time. This can be used on a single element with a pseudo class to provide visual feedback that uses motion.

A canonical example: a drop down menu that slides down when activated by hovering over another element.

Where else could we use transitions?

Components of Transitions

  • Animatable propertys
  • An initial state
  • A final state
  • A duration
  • A delay (optional)
  • A timing function (optional)

Performance

CSS Transitions don't just happen... The browser must re-render and reflow layouts constantly during an animation.

There are some properties we can use that incur the least performance penalty, these are CSS Transforms

Transforms

CSS Transforms allow us to change the location, scale, rotation and opacity of an element. These correspond to translate, scale, rotation and opacity.

These transitions are computed on the GPU and a seperate layer so they are fast and don't force your layouts to reflow.

We control transforms the same way we control any other CSS Animatable Property transition

Animation

The Animation Property

In CSS we can animate several properties using the animate property. This also provides us the ability to specify keyframes and create more complex animations.

A keyframe in traditional animation is a frame in the animation sequence that marks a big change. Think of this as a car turning a corner, we need a keyframe at the corner to make the turn.

Components of the Animation Property

  • Name
  • Duration
  • Timing Function (optional)
  • Delay (optional)
  • Direction (optional)
  • Iteration Count (optional)
  • Fill Mode (optional)
  • Play State (optional)

Animation - Example

Differences Between Animations and Transitions

Transitions specify the animation that should be applied to a change in a specific or multiple CSS properties, while Animations provide a framework for named animations that can alter multiple CSS properties and provides a keyframing system that can change properties multiple times over the course of one animation.

Performance

CSS Animations suffer from the same performance issues as Transitions.

Likewise, CSS transforms incur the least performance penalties.

Which do I use where?

First off... be sensible about your use of both Transitions and Animations, why are you using them and where? Does it help your user?

For this project, transitions should be sufficient.

If you are creating something more complex, you may want to practice using animations.
