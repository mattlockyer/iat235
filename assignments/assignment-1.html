Assignment 1: Information Architecture

Description

Working with our Project 1 materials from prior weeks you will be building an information architecture using a couple different approaches to understand what will be needed from your website.

Due

Before LAB 04

Worth

5% of your final grade

Instructions

  1. Create an information architecture that illustrates the use case ofposting an item/event/activity. This should be a top-down structure that includes:

    • exploring what is needed from the use case (from the user's side)
    • covering how a user might begin to approach it or understand it
    • not about building a sitemap, but organizing task information into linked categories
    • think about the user and how they are meant to flow through the task

    Example: a user has posted a single image, where do they return to after?

  2. Create an information architecture that illustrates the structure of the section you are posting in This should be a bottom-up structure.

    • Sub Categories
    • Search
    • Posting
    • Navigation Options
    • ...

    Include the item detail, every possible component and how this is organized

    • Title
    • Subtitle
    • Location
    • Map
    • Images
    • ...

Final Delivery - TO BE HANDED IN

Final deliverables are due to Canvas before your lab time on Week 4. Please remember that professionalism in your submission is important:

  • Create 1 final document using a vector graphics program (i.e. Draw.io or Illustrator) that shows the top-down and bottom-up architectures separately. We are looking for clean, clear and effective presentations of both architectures.

Grading Rubric

Each requirement receives points as outlined:

  • Effective top-down information architecture (1.5 pts)
  • Effective bottom-up information architecture (1.5 pts)
  • Design quality (2 pts)
