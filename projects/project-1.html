Project 1: Scope

Description

This term we will be working on designing a section of Craigslist Vancouver to improve it's information and web design. For your first project, we will be spending some time defining the context and requirements of the upcoming P02 and P03.

This project is to be completed individually.

Due

Before your LAB 03 (June 3rd).

Worth

5% of your final grade.

Instructions

There are a number of weeks in which to complete this project.

Week 1

Choose one of the main sections of Craigslist Vancouver (community, housing, jobs, for sale, etc... NO PERSONALS) and spend some time answering the following questions. Focus on (a) finding information and (b) posting information.

  1. Purpose:
    • What does it appear this portion of the website was built to do?
    • Who is it meant for?
    • How did you conclude who and what it was meant for?
  2. Information:
    • Create a diagram of the hierarchy of content
    • How clear is the hierarchy of information on this page? and why?
    • How is this information presented?
  3. Use:
    • Determine ALLLLL methods of navigation to find an item/activity/event in your home area
    • How easy or clear was it to navigate using these methods? and why?
    • How consistent are the design and methods of navigation?

Organize #1, 2, and 3 from above into an easily understood document. You will be discussing your findings next week in the labs, so it is important that you make it readable for your classmates.

Week 1 deliverables (due in your Week 2 lab):
  • 1 completed set of answers in a readable format

Week 2

  1. In the lab there will be crits, so make notes as necessary.

  2. For the use case of posting an item/event/activity create a list of functional specifications (minimum 3), remember these functional specifications should be:

    1. Positive: You shouldn't deny, but instead direct
    2. Specific: Avoid allowing any ambiguity
    3. Objective: Use non-subjective language

  3. Create a summary of content requirements

    • What information will a user need to be able to complete the task?
    • What information will the system need to be able to complete the task?
    • What is the purpose of that information?
    • How would that information be best presented?

Final Delivery - TO BE HANDED IN

Final deliverables are due to Canvas before LAB 03 (June 3rd), and double-check all your submitted files to ensure they can be opened. All deliverables should have your name on it.

1 summary document (PDF), which includes:

  1. Create an analysis summary which clearly outlines the information provided, the purpose for and how the site is intended to be used using evidence from your research. Evidence can include sketches, screenshots or explanations as you see fit. Use complete sentences; no point form is allowed.

    (max 500 words)

  2. Create a scope summary which clearly outlines the functional and content specifications for posting an item, using examples from your research. Examples can include sketches, screenshots or explanations as you see fit. Use complete sentences; no point form is allowed.

    (max 500 words)
